GITHUB

¿Qué es GitHub?

GitHub es un repositorio online gratuito mediante el cual poder gestionar proyectos y controlar versiones de códigos. Especialmente útil para desarrolladores.

GitHub también es utilizado para hospedar proyectos de código abierto, realizar un seguimiento de fallos, revisar el código, compartirlo...

Iniciarse en GitHub

Para comenzar en GitHub necesitaremos crear una cuenta. Para ello visitaremos su pagina oficial: https://github.com

A continuación elegiremos un nombre de usuario, una contraseña y un correo, al cual deberemos acceder para revisar todas nuestras notificaciones.

Ajustes de la cuenta

Una vez confirmado nuestro correo electrónico y habiendo iniciado sesión en GitHub nos dirigiremos a nuestro perfil, en el podremos modificar nuestro icono, añadir direcciones de correo secundarias, activar la autentifican en 2 pasos y más.

Participar en proyectos

Con nuestra cuenta puesta a punto ya podremos comenzar a participar en proyectos.

Para poder participar en un proyecto ya existente primero deberemos hacer un "fork", esto nos dará una copia del proyecto, la cual podremos modificar.

Pull Request

Si tras modificar el código original de una persona mediante el fork creemos que lo hemos mejorado podemos realizar un pull request.

Un pull request es una solicitud de implementación de código al código de otra persona, el cual hemos editado con nuestro fork.

Una vez realizamos el pull request al autor original se le notificara por correo, y tras revisar los cambios puede decidir si aplicarlos a su código, rechazarlos o ignorarlos.

Crear un repositorio

En GitHub ademas de beneficiarnos del código de otras personas podemos crear nuestro propio repositorio, un lugar de almacenamiento virtual para nuestros proyectos.

Estos repositorios también pueden ser compartidos copiando la URL del mismo, ya sea HTTP o SSH y compartiéndola con quienes nosotros queramos.

Ahora que tenemos nuestro propio repositorio y algo de código puede que algunas personas se interesen y lo modifiquen, con lo cual nos harán un "pull request".

Cuando alguien realiza un pull request en nuestro código se nos notificara por correo electrónico. El contenido del correo constara de una lista de archivos modificados, un comando para que fusionemos el código. Ademas podremos entablar una discusión con la persona que realiza el pull request para comentar los cambios realizados.

Si al final optamos por realizar los cambios podremos realizarlo mediante el comando "git pull -url- -branch-", o bien de manera gráfica pulsando el botón "Merge" en GitHub

Colaboradores

Como ya has visto GitHub permite que varias personas interactúen con el mismo código, por eso existe la figura de los colaboradores, personas con las cuales trabajaras en el mismo proyecto y repositorio y que tendrán permiso de lectura y escritura sobre el código, de esta manera no sera necesario realizar un pull request cada vez que se modifique el código, a este permiso se le denomina "push".

Para añadir colaboradores simplemente deberás desplazarte al repositorio al cual desees añadirlos y navegar por el menú de ajustes > Gestionar acceso > invitar a colaboradores. Al clickar en este botón deberemos introducir sus nombres de usuario de GitHub o su correo electrónico, de cualquiera de las dos formas los colaboradores deberán responder a la solicitud a través de una notificación en el correo electrónico.

Oragnizaciones

Del mismo modo que se pueden añadir colaboradores a un repositorio se puede crear una organización propia.

Las organizaciones son de gran utilidad, ya que se puede ordenar mejor los grupos de trabajo y formar distintos equipos para repartir el trabajo.

Para la creación de una organización debemos darle click al símbolo + al lado de nuestro icono y seleccionar "nueva organización". Ahora deberemos darle un nombre y añadir una dirección de correo electrónico de contacto.

Dentro de las organizaciones se pueden crear multitud de repositorios y bifurcarlos.

Equipos

Los equipos son agrupaciones de gente dentro de una organización con acceso a diferentes repositorios.

Para crear un equipo basta con dirigirnos a la organización en la que queramos crear el equipo y pulsar en la barra "Equipos", aquí podremos añadir a las personas que formaran dicho equipo, los repositorios a los que tendrán acceso y modificar los equipos ya creados y sus accesos.
