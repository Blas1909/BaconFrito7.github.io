GIT LOCAL

¿Qué es git?

Git es una software de control de versiones lanzado en 2005 para mejorar la eficiencia, la fiabilidad y compatibilidad de versiones de aplicaciones cuando tiene un gran numero de archivos.

Mediante Git podemos crear repositorios, lugares de almacenamiento virtuales para nuestros proyectos. En estos repositorios podremos guardar las diferentes versiones de código que tengamos y acceder a ellas.

Tener en cuenta que este apartado es mediante consola.

Repositorios en Git

Para la creación de un repositorio utilizaremos el comando de terminal "git init".

También podemos clonar un repositorio ya existente y realizar aportes al mismo, para clonar un repositorio utilizaremos el comando "git clone".

La carpeta repositorio es un elemento oculto, por lo que para poder visualizarlo en nuestro equipo tendremos que tener activada la casilla que nos permita ver elementos ocultos en el explorador de archivos.

Primeras adiciones a nuestro repositorio

Una vez creado nuestro repositorio ya podremos comenzar a guardar las diferentes versiones de nuestros proyectos haciendo uso del comando git -add "nombre del archivo". Pero con esto no basta para añadir nuestros archivos ya que de momento los añadirá al área de ensayo, para confirmar su subida utilizaremos el comando "git commit -m 'comentario sobre el commit'".

El estado del repositorio

Para comprobar el estado de nuestro repositorio podemos utilizar multitud de comandos:

El comando "git status" nos muestra el estado del área de trabajo, es decir los archivos a los que hayamos hecho git add, pero aun no hemos hecho commit.

El comando "git log" nos muestra el historial completo de todo lo que ha sucedido a nuestro repositorio.

El comando "git diff" nos muestra los cambios exactos entre algunos archivos. Por defecto este comando nos mostrara las diferencias del ultimo commit, pero también podemos comparar 2 archivos en concreto haciendo uso de su id, el cual son los 6 primeros caracteres.

Las ramas

A la hora de trabajar con git también podremos utilizar distintas ramas, dichas ramas son nuevas áreas de trabajo, con su propio historial y área de ensayo. Por defecto la rama principal se llama "master".

Creacion de ramas

Para crear una nueva rama utilizaremos el comando "git branch 'nombre de la rama'".

Movernos entre ramas

Si ahora deseamos ir moviéndonos por las diferentes ramas haremos uso del comando "git checkout 'nombre de la rama'".

Fusión y eliminación de ramas

Supongamos ahora que queremos fusionar las distintas ramas de nuestro repositorio, en primer lugar nos situaremos en la rama principal utilizando el ya mencionado comando "git checkout 'nombre de la rama'".

Si no sabemos en que rama nos encontramos podemos hacer uso del comando "git branch", tras ejecutarlo se nos mostraran todas las ramas existentes, pero una de ellas tendrá un "*" delante, esa sera la rama en la que nos encontremos actualmente.

Una vez situados en la rama deseada usaremos el comando "git merge 'nombre de la rama a fusionar'".

Pero ahora nos queda una rama vacía, así que la eliminaremos usando el comando "git branch -d 'nombre de la rama a eliminar'".
